java方法参数传递方式

经常看到有人在网上讨论Java方法到底是值传递还是引用传递。其实这个问题没什么难的，我们不知道有时候只是因为我们不愿意知道，或者是没有用心思考和记忆，同样我也是，希望这次是最后一次回顾这个问题。

首先说明一下Java方法参数传递是值传递

什么是值传递，什么是引用传递？

值传递

在参数传递的时候，方法的参数实际上相当于方法内的局部变量，在我们传递参数的时候，就相当于给方法内的局部变量赋了值。

所以在内部对这个局部变量重新赋值或者重新指向一个对象，在方法执行完毕之后，对原来的变量没有影响。

值传递就是对传的参数的一个拷贝，无论这个参数是一个引用还是基本数据类型。

所谓的值传递就是

引用传递

相信学习过C++的同学对这个说法都不会陌生，我们在学习C语言的时候，遇到过一个经典的问题：怎么用一个函数交换两个数值。那时候的解决方法就是传递指针。

引用传递就是传递参数本身的地址。

但是值传递为什么可以改变对象属性的值

Java中如果传递的参数是对象的引用，那么在方法中的形参其实该引用的一个副本，两个引用指向的是同一个对象，如果在方法中将该引用的副本指向另外一个对象，对原对象是没有任何影响的，但是如果对该对象的属性进行操作还是起作用的，毕竟指向的是同一个对象。
