异常

有疑问，别存着。
很久之前就搞不明白Checked异常，RuntimeExctption，不动throw和throws啥区别，今天终于搞明白了

Checked异常

Checked异常是指可以被处理修复，而且程序员通过代码无法避免的异常，比如SQLException、IOException，必须显示的处理这种异常，否则无法通过编译。

Runtime Exception

编译阶段不用必须处理，程序员可以通过代码逻辑避免的，比如IndexOutOfBoundsException。不用必须在代码中显示处理。

throws

throws是在方法签名的时候抛出异常。当方法不知道怎么处理该异常的时候，可以通过throws将异常抛给上一级处理。该方法的调用者可以处理该异常，也可以再抛给上一级。

throw

throw在方法体中抛出异常，Checked异常和Runtime异常有区别

如果是抛出Checked异常，则必须在该方法体中显示的处理或者通过throws抛给上层；如果是RuntimeException,可以不处理，也可以按照上面的方式处理。
